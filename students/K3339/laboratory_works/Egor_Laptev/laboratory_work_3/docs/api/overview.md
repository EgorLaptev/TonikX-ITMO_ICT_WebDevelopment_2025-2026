# Обзор API Endpoints

Все endpoints используют стандартные HTTP методы для выполнения операций CRUD (Create, Read, Update, Delete).

## Список доступных endpoints

| Endpoint | Описание | Модель |
|----------|----------|--------|
| `/api/room-types/` | Управление типами номеров | RoomType |
| `/api/floors/` | Управление этажами | Floor |
| `/api/rooms/` | Управление номерами | Room |
| `/api/guests/` | Управление гостями | Guest |
| `/api/stays/` | Управление проживаниями | Stay |
| `/api/employees/` | Управление сотрудниками | Employee |
| `/api/cleaning/` | Управление расписанием уборки | CleaningSchedule |

## Стандартные операции

Все endpoints поддерживают следующие операции:

### GET `/api/{resource}/`
Получить список всех объектов.

**Ответ:** 200 OK
```json
[
  {
    "id": 1,
    ...
  }
]
```

### GET `/api/{resource}/{id}/`
Получить конкретный объект по ID.

**Ответ:** 200 OK
```json
{
  "id": 1,
  ...
}
```

### POST `/api/{resource}/`
Создать новый объект.

**Тело запроса:** JSON объект с полями модели

**Ответ:** 201 Created
```json
{
  "id": 1,
  ...
}
```

### PUT `/api/{resource}/{id}/`
Полностью обновить объект (все поля обязательны).

**Тело запроса:** JSON объект со всеми полями модели

**Ответ:** 200 OK
```json
{
  "id": 1,
  ...
}
```

### PATCH `/api/{resource}/{id}/`
Частично обновить объект (только указанные поля).

**Тело запроса:** JSON объект с полями для обновления

**Ответ:** 200 OK
```json
{
  "id": 1,
  ...
}
```

### DELETE `/api/{resource}/{id}/`
Удалить объект.

**Ответ:** 204 No Content

## Коды ответов

| Код | Описание |
|-----|----------|
| 200 | Успешный запрос |
| 201 | Ресурс успешно создан |
| 204 | Ресурс успешно удален |
| 400 | Неверный запрос (ошибка валидации) |
| 401 | Требуется аутентификация |
| 403 | Доступ запрещен |
| 404 | Ресурс не найден |
| 500 | Внутренняя ошибка сервера |

## Формат запросов и ответов

Все запросы и ответы используют формат JSON.

**Заголовки запроса:**
```
Content-Type: application/json
Authorization: Token {your_token}
```
